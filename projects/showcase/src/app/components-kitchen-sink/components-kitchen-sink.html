Buttons

Normal: intent omitted (default) for routine, non-destructive actions.

Primary: intent="primary" for the highest-emphasis action in a view, usually one per section.

Secondary: intent="secondary" for alternate actions when primary already represents the main path.

Quiet: intent="quiet" for low-emphasis actions that remain discoverable without stealing focus.

Danger: intent="danger" for destructive actions; pair with clear labeling and confirmation when irreversible.

Icon action buttons

Icon-only: use shape="round" size="sm", prefer intent="action" for utility controls, and always provide an accessible name via aria-label. Keep SVGs decorative with aria-hidden="true".

Badge

Badge is a status indicator or notification counter. Sits absolutely on or near another element.

Default (notice): <super badge [count]="3"></super> — subtle blue badge for general notifications.

Warning intent: intent="warning" for caution states.

Danger intent: intent="danger" for urgent or destructive contexts.

Success intent: intent="success" for positive/completed states.

Overflow: counts over 99 show 99+ automatically.

Indicator only (empty): <super badge intent="success" state="pulse"></super> — status dot with pulse animation.

Position variant: position="bottom-left" for corner placement (default: top-right).

Icon button with badge

Pattern: badges on icon buttons use position="top-right" (default) to sit outside the circular hit area.

Tab with unread count

Pattern: badges in tab bars show activity counts; reuse intent colors for consistency with button intents.

Dialog

Dialog is the first floating-layer primitive. It uses the native <dialog> element for semantics and keyboard behavior, while Angular manages open state and focus return.

Last close result: {{ dialogResult() }}

Backdrop: frosted glass comes from theme tokens such as --dialog-backdrop-color and --dialog-backdrop-blur, so schemes can soften or intensify it without rewriting component CSS.

Dismissal default: backdrop/light-dismiss is opt-in via [closeOnBackdrop]="true"; default behavior keeps modal dialogs open unless action buttons or Escape close them.

Implementation note: if an app restyles the host dialog[se] as the full visual surface while keeping target-equality backdrop dismissal, clicks on host-hit areas near content edges can dismiss unexpectedly. Keep a dedicated inner surface boundary or adjust dismissal logic for host-only layouts.
